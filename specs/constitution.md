# cxg Constitution

## Core Principles

### I. Simplicity First
Make every change as simple as possible. Impact minimal code.

### II. No Laziness
Find root causes. No temporary fixes. Senior developer standards. Don't fear breaking changes.

### III. Minimal Impact
Changes should only touch what's necessary. Avoid introducing bugs.
